# GLOBAL — Common Brain 헌법

> **Common Brain OS v1.0**
> 전 조직(cozybuilder)에 적용되는 표준과 규칙의 **단일 소유 문서**.
> 여기 적힌 내용이 다른 문서/프로젝트와 충돌하면 **이 문서가 우선**한다.

이 문서는 자주 바뀌지 않는다. 변경은 곧 표준의 변경이며, 반드시 커밋으로 남긴다.

---

## 1. 핵심 원칙 (전체 정의)

### 1.1 Document First, GitHub Second
- 모든 운영 결정은 **문서에 먼저** 기록한 뒤 코드/저장소 작업을 한다.
- 문서가 원본(source)이고, GitHub의 상태는 문서를 반영한 결과(reflection)다.
- "코드를 먼저 짜고 나중에 정리한다"를 금지한다. 결정 → 문서 → 실행 순서를 지킨다.

### 1.2 Single Source of Truth (SSOT)
- 모든 사실은 정확히 **한 곳에만** 존재한다.
- 같은 정보를 두 문서에 적지 않는다. 필요하면 **복제 대신 링크**한다.
- 정보의 소유 위치는 아래 1.3 문서 소유권 표를 따른다.

---

## 2. 문서 소유권 (Ownership Map)

각 정보는 아래에 명시된 문서만이 소유한다. 다른 문서는 이를 인용/링크만 한다.

| 정보 종류 | 소유 문서 |
|-----------|-----------|
| 핵심 원칙 · 표준 · 규약 | **GLOBAL.md** (이 문서) |
| 운영 절차 · 워크플로 | **PLAYBOOK.md** |
| 프로젝트 목록 · 메타데이터 · 상태 포인터 | **PROJECTS.md** |
| 개별 프로젝트의 살아있는 상태 | 각 프로젝트의 **STATUS.md** (PROJECTS.md가 포인터를 소유) |
| 신규 프로젝트 골격 | **templates/project-template/** |
| 소통·보고 원칙 | **GLOBAL.md §7** (이 문서) |
| 보고 형식 템플릿 | **PLAYBOOK.md §P3** |

## 3. 저장소 규약

- **GitHub Org**: `cozybuilder`
- **기본 브랜치**: `main` (신규 저장소는 `main`으로 생성한다)
- **저장소 명명**: 소문자-케밥(kebab-case) 권장. 기존 저장소는 현 이름을 존중한다.
- 원격 연결은 결정·문서화 후에 한다 (Document First).

## 4. 커밋 규약

- 형식: `<type>: <요약>` (예: `docs: add PLAYBOOK onboarding flow`)
- type 예시: `docs`, `feat`, `fix`, `chore`, `refactor`
- 한 커밋은 하나의 논리적 변경만 담는다.
- 운영 문서 변경은 `docs:` 를 사용한다.

## 5. 명명 규약

- 문서 파일: 루트 표준 문서는 **대문자**(`README.md`, `GLOBAL.md`, `PLAYBOOK.md`, `PROJECTS.md`).
- 그 외 일반 문서/디렉터리: 소문자-케밥.
- 프로젝트 식별자(ID): PROJECTS.md 등록부의 이름과 정확히 일치시킨다.

## 6. 결정 기록 원칙

- 의미 있는 운영 결정은 그 결정을 소유하는 문서에 반영하고 커밋한다.
- "왜"가 필요한 결정은 본문에 근거 한 줄을 남긴다.

## 7. 소통·보고 원칙

- 클로(Claude) → 코비(운영자) **완료 보고는 항상 복붙 가능한 블록 안에** 작성한다. 일반 평문 보고는 **금지**한다.
- 이 규칙은 **새 대화창에서도 유지된다** — 본 문서가 그 단일 근거(SSOT)다. 세션 기억이 아니라 이 문서를 따른다.
- 보고 본문의 구조(템플릿)는 [PLAYBOOK.md §P3](PLAYBOOK.md)가 소유한다 — 여기서는 원칙만 정의한다.

## 8. OS 버전 관리

- 이 운영 체계의 버전은 **Common Brain OS v1.0**.
- 구조·원칙의 비호환 변경 시 메이저 버전을 올린다(v2.0). 문서 추가/보완은 마이너로 본다.
